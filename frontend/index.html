Testing Guide

How to test and interpret predictions from the Abstention Classifier.

1

Understanding the Features

The model expects 30 numerical features per transaction:

  • Time — Seconds elapsed since the first transaction in the dataset (range: 0 to ~172,000).
  • V1 through V28 — Principal components obtained via PCA transformation for anonymization. These typically range between -5.0 and +5.0, though fraud samples may have extreme values up to +/-30.
  • Amount — The monetary value of the transaction in Euros (range: 0 to ~25,000).
2

Using Sample Data

Click the sample data buttons above the form to auto-fill with realistic test vectors:

  • Legitimate — Standard transaction with normal PCA values. Expect a high confidence Legitimate prediction.
  • Fraud-like — Values with large deviations in V1-V28 and a low Amount. Should trigger a Fraud prediction.
  • Uncertain — Borderline features where multiple classes have similar probabilities. The model may choose to Abstain.
3

Interpreting Results

After prediction, examine three outputs:

  • Prediction Label — Legitimate (class 0), Fraud (class 1), or Abstain (class 2).
  • Confidence Scores — Softmax probabilities for each class. Higher means the model is more certain.
  • Decision Status — "Should Decide" indicates the model committed to its prediction. "Should Defer" means the model abstained and recommends human review.
4

Experimentation Tips

To explore model behaviour:

  • Start with a sample, then modify individual features to observe how predictions shift.
  • Try extreme values (e.g., V14 = -15.0 or Amount = 10000) to push the model towards fraud.
  • Gradually increase uncertainty by setting V1-V5 values close to zero — the model may start abstaining.
  • Compare the confidence distribution in the doughnut chart for each scenario.
